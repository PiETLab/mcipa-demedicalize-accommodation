# The ActionForge Project concept: GitHub as public infrastructure for collective work

This project uses GitHub as a practical way to publish, organize, preserve, and share the materials of a community-based initiative.  GitHub is most often used as a software-development tool, but this project is about subverting its use for our needs.

We call the project “ActionForge” as a play on the term "software forge" which refers to an integrated environment that supports collaborative software development.  Instead of forging software source code, ActionForge helps grassroots groups forge the materials, records, and public infrastructure that support collective action.

For many people, GitHub sounds like the wrong place for activism or grassroots work. It is usually associated with programmers and code. But GitHub can also function as a public record-keeping and publishing system. It can host ordinary text pages, reports, resource lists, project explanations, decisions, and websites. It can also show how materials change over time.

The basic idea is that a project can have a durable public home. Instead of relying only on scattered emails, private folders, social media posts, or institutional websites, the project keeps its materials in one structured place. Those materials can be read, linked to, copied, improved, and maintained over time.

This matters for activism, community research, mutual aid, accessibility work, and other initiatives “from below” because these groups often need low-cost infrastructure that they can control. They need ways to explain their work, document what they have learned, preserve collective memory, and make resources available beyond a single event, grant, or campaign.

The goal is not to turn activists into software developers. GitHub is not a substitute for relationships, organizing, meetings, care, or political work. It is simply supporting infrastructure: a place where the work can be made visible, findable, and durable.

This project asks what becomes possible when grassroots initiatives use the same durable tools as software communities, but for public knowledge, advocacy, documentation, and collective memory.
