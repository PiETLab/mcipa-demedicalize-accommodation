# GitHub Account Setup (Sabine)

## Audience

Content Operator (Sabine) and Infrastructure Steward (Melanie).

## Why this is needed

Repository access and routine pushing cannot be configured until Sabine has an active GitHub account.

## Steps to create a GitHub account

1. Go to `https://github.com/signup`.
2. Use an email address you will keep long-term for project work.
3. Choose a username suitable for professional collaboration.
4. Create a strong password (use a password manager if available).
5. Complete email verification from GitHub.
6. Sign in and complete any required onboarding prompts.

## Immediate post-signup setup

1. Enable two-factor authentication (2FA) in account security settings.
2. Add a profile name so collaborators can identify your account.
3. Confirm you can sign in from your normal working device.

## Handoff to Melanie

After account creation, send Melanie:

- your GitHub username
- confirmation that email is verified
- confirmation that 2FA is enabled

Melanie will then grant repository access in PiETLab and complete repository access configuration.

## How to confirm repository access was granted

Use at least one of these checks:

1. You can open the repository page while signed in and do not see a request-access/404 denial state.
2. You receive and accept a repository/collaborator or organization invitation from GitHub (if sent).
3. A pull test from Obsidian Git or Git client succeeds without permission errors.

If access is still unclear, ask Melanie to confirm your username has been granted the intended repository permissions.

## Completion criteria

- [ ] GitHub account created
- [ ] Email verified
- [ ] 2FA enabled
- [ ] Username shared with Melanie
- [ ] Repository access confirmed
