# Have a Look Around Your New Obsidian Vault

## Purpose

Orient yourself inside the newly opened vault before plugin setup and Git connection work begins.

## Audience

Content Operator (Sabine).

## Preconditions

- The cloned project folder already exists.
- The vault is open in Obsidian.
- You have not yet started Obsidian Git plugin setup.

## Instructions

1. Open the vault in Obsidian, confirm the vault name is correct.
2. Look at the folder tree. Locate the `content` folder - that is the content for the website `https://pietlab.github.io/mcipa-demedicalize-accommodation/`
3. Open the note `index.md`. In your browser,  open `https://pietlab.github.io/mcipa-demedicalize-accommodation/`.   Notice how the `markdown` content in `index.md` has been transformed into  `html` for the website.  That was accomplished by the Quartz site generator software.
4. Do not change content yet; this step is only for orientation.

## Verify success

- Vault opens normally in Obsidian.
- Folder tree is visible and matches expectations.
- You know where the main working areas are located.
- No unexpected changes were made.

## Escalate when

- The vault does not open cleanly.
- The folder tree is missing or empty in an unexpected way.
- You cannot tell whether you opened the correct vault.
